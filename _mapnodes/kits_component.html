{{ page.title }}

Game kits that control what items or player aspects should be apply to each kit. This is a very advance system that is very customisable. You may not name any kit default that is the internal kit name that reset's the player and is given is the player does not have a kit assigned to them.

Field Type Description Default
gamemode String The gamemode the player will be in survival, creative, adventure, spectator. Survival
fly Boolean Will players be able to fly like creative mode. False
health Integer The player's health level must be bigger than 1. 20
food Integer The player's food level, must be 0-20 20
permissions List The list of permission nodes the player will recive. None
armor Armor The item to be used for the player's helmet. None
items List The items the player's will be granted. None
effects List The potion effects the player's will be granted. None

Armor

Armor defines the helmet, chestplate, leggings, and boots. Their values are items.

Field Type Description Default
helmet Item The item to be used for the player's helmet. None
chestplate Item The item to be used for the player's chestplate. None
leggings Item The item to be used for the player's leggings. None
boots Item The item to be used for the player's boots. None

Item

Items what makes up an item the slot, type, amount, damage value, and nbt data.

Field Type Description Default
slot Integer The slot position on the player's inventory. None
item String The name of the item type. Required
damage Integer The damage value of the item. 0
amount Integer The amount of items that will in a slot. 1
nbt NBT The items nbt data. None

NBT

NBT is extra data to the item to make it diffrent from other items with the same damage value. You can apply any nbt data to any item though you may not get expected results.

Field Type Description Default
display Display Controls the display of the item. None
enchantments List Add enchants to the items. None
title String The title of the book. None
author String The author of the book. None
pages List Each element of the list is a new page. None

Display

Display control the display of the item. This will change the name, lore, or color. The color value will only work on leather armor. Currently they only accept a chat color but is planed to suport hex color codes. If the leather armor is not colored it will default to the teams color.

Field Type Description Default
name String The name of the item. None
lore List A list of lore line the item will have. None
color ChatColor The color value of the leather armor. None
{% highlight json linenos %} { "name": "&6Gold Apple", "lore": ["Line one", "Line two"], "color": "white" } {% endhighlight %}
Enchantment

Enchantment is magic that is applied to the item to make it better.

Field Type Description Default
name String The name of the enchanment. Required
level Integer The level of the enchantment. Required
{% highlight json linenos %} {"name": "arrow_damage", "level": 1} {% endhighlight %}
Potion Effects

Potion Effect are effects that are applied to the player.

Field Type Description Default
name String The name of the Potion Effect. Required
duration Duration The time lenght of the effect, use -1 to make it forever. Required
level Integer The potion effect level that will be applied. 1
ambient Boolean Are the particle translutant like they apear near beacons. false
show_particles Boolean Show particle around the player. true
